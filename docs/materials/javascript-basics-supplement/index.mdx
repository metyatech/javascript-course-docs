## 今後 Web 開発関連で学ぶこと

### TypeScript

**JavaScript** に多数の機能を追加してより便利にしたプログラミング言語。(厳密なスーパーセット)
JavaScript の書き方でそのまま書いても動作する。
昨今 **JavaScript** ではなく **TypeScript** で開発するところも多い。
(ブラウザでは結局 JavaScript しか動作しないので) **JavaScript** に変換して実行する。(トランスパイル)

TypeScript の例

```ts
// TypeScript: 型を明示できる
function add(a: number, b: number): number {
    return a + b;
}

let x: number = 10;
let y: number = 5;

// x = "こんにちは"; // ← エラーになる（型が違う）

console.log(add(x, y)); // 15

// 配列の型も指定できる
let numbers: number[] = [1, 2, 3];
// numbers.push("4"); // ← エラーになる（型が違う）

console.log(numbers);
```

#### Webアプリの「構造」の違い

##### シングルページアプリケーション（SPA）
- HTMLは基本的に1つ
- 画面の切り替えは JavaScript が行う
- ページ遷移しているように見えるが、実際は同一ページ内の更新

- 例）
  Facebook、X、YouTube、Teams（Web版）、
  Gmail、Google Drive / Document / Spreadsheet / Slide / Calendar、
  Microsoft 365 for the Web

##### マルチページアプリケーション（MPA）
- HTMLが複数存在する
- 画面の切り替えをHTMLの切り替えで行う
- 昔からある「一般的なWebサイト」の構造

## 生成AIとは

**生成AI**

- **生成AI**とは？
- テキスト、音声、画像、動画など、新しいコンテンツを生成することのできるAIのこと。
- 例）ChatGPT, Microsoft Copilot, Gemini, Adobe Firefly, AI Overview, Midjourney

## AIモデルの基本

**AIモデル**

- **AIモデル**
  - 特定のタスクを実行するために設計された数学的な構造
  - 一般的に、「入力→**AIモデル**→出力」というプロセスで動作する
- 例）
  - GPT (Generative Pre-trained Transformer)
  - Gemini
  - Claude
  - → これらは、いずれも、入力：文章　出力：次に来るであろう単語とその確率のリスト

**入力と出力の例**

| 入力 | 出力 |
| --- | --- |
| 今日は | (天気,35%), (何を,20%), (良い, 15%), (仕事, 10%), (どうする, 10%),… |
| 日本一 | (高い,30%), (美しい,20%), (の山,15%), (寒い,10%),… |
| 世界で | (一番,35%), (最も,25%), (有名,15%), (広い,10%), (強い,10%),… |
