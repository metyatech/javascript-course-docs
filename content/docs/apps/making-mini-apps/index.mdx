---
title: ミニアプリの作り方
---

これまでに学んだ JavaScript（変数・条件分岐・繰り返し・配列・オブジェクト・関数・DOM 操作・イベント）を組み合わせて、ボタンで動く小さなアプリを作っていきます。このページでは、どのミニアプリにも共通する「進め方」と「つまずいたときの確認ポイント」をまとめます。

## 作る順番

新しいアプリは、いきなり完成形を目指すより、次の順番で進めると迷いません。

1. **まず動かす** — ボタンを押すと何かが起きて、エラーが出ない状態をまず作る。
2. **次に中身を変える** — 文章・結果・問題・選択肢を、自分のテーマに書き換える。
3. **最後に見た目を変える** — 余裕があれば、色・余白・文字サイズなどの CSS を調整する。

最初から完璧な見た目にする必要はありません。「動く雛形を、自分のテーマに改造する」と考えると進めやすくなります。

## つまずいたときの確認

:::caution ボタンを押しても何も起きない
- JavaScript ファイルが読み込まれているか
- ボタンの id と `querySelector` の文字が一致しているか
- `addEventListener` のスペルは正しいか
- 関数を渡すとき、うしろに `()` を付けていないか
:::

`addEventListener` には「関数そのもの」を渡します。うしろに `()` を付けると、その場で実行した結果が渡ってしまい、ボタンを押しても動きません。

```js
// 正しい：関数そのものを渡す
button.addEventListener('click', showResult);
```

```js
// まちがい：呼び出した結果を渡している
button.addEventListener('click', showResult());
```

:::caution 文字が変わらない
- `textContent` で書き換えているか
- 変えたい要素を `querySelector` で取得できているか
- 変数を変えただけになっていないか（画面の表示も更新する必要がある）
:::

:::caution 値が undefined になる / 結果が出ない
- 配列の番号は 0 から始まる（存在しない番号を指定していないか）
- オブジェクトのプロパティ名が一致しているか（`item.name` など）
:::

ランダムな番号は、配列の長さをかけてから `Math.floor` で小数を切り捨てると、`0` から `(長さ - 1)` の範囲で作れます。

```js
const index = Math.floor(Math.random() * items.length);
```

:::tip どこを直せばいいか分からないとき
1. まずコンソールのエラーを見る（一番上のエラーから直す）
2. `console.log` で変数の中身を確かめる
3. 最後に変更したところを疑う
4. 動いていた状態まで戻す
:::
