## 条件分岐とは？

条件によって実行する処理を切り替えることを**条件分岐**といいます。
「点数が80点以上なら合格メッセージを表示する」「入力値によって返答を変える」といった場面で使います。

---

## if 文

条件が **真（true）** のときだけ、波括弧 `{}` 内の処理を実行します。

```js
if (条件) {
    // 条件が true のときの処理
}
```

<CodePreview>
```javascript
let answer = prompt('日本一高い山は？');

if (answer === '富士山') {
    console.log('正解！'); // answer が "富士山" のときだけ出力される
}
```
</CodePreview>

### if文のQuickCheck

<QuickCheck>

次のコードでは何が表示されますか？

```javascript
let answer = 'Python';

if (answer === 'JavaScript') {
    console.log('正解');
}
```

<Hint>
`answer === 'JavaScript'` が `true` と `false` のどちらになるか確認しましょう。
</Hint>

<Answer>
何も表示されません。

`answer` には `'Python'` が入っているため、条件は `false` です。波括弧の中の処理は実行されません。
</Answer>

</QuickCheck>

## JavaScriptのインデント

JavaScriptでは、**波括弧 `{ }` で囲まれたブロックの中**をインデントします。
インデントを適切に設定しないと、どこからどこまでが一つのまとまり（ブロック）なのかが分からなくなり、エラーの原因になります。

:::note
インデントの基本的な考え方は、次の共通基礎で確認できます。

- [入れ子とインデント](https://web-foundations-docs.vercel.app/docs/nesting-indentation)
:::

### 良い例（インデントが揃っている）

```javascript
if (条件) {
    // 波括弧の中なのでインデントを1段下げる
    console.log('真です');
}
```

### 悪い例（インデントがぐちゃぐちゃ）

{/*prettier-ignore*/}

```javascript
if (条件) {
console.log("真です");
}
```

悪い例では、コードがどの波括弧に含まれているのかが視覚的に分かりづらくなっています。プロの現場ではインデントが揃っていないコードはそれだけで「読みにくい」と評価されてしまいます。VSCodeではコードを選択して `Alt + Shift + F` (Windows) または `Option + Shift + F` (Mac) で自動的に綺麗に揃える機能（フォーマット）もありますが、まずは自分で「波括弧の次は下げる」意識を持ちましょう。

## if...else 文

条件が **真のとき** と **偽のとき** で、それぞれ別の処理を実行できます。

```js
if (条件) {
    // true のときの処理
} else {
    // false のときの処理
}
```

<CodePreview>
```javascript
let input = prompt('helloを入力して下さい');

if (input === 'hello') {
    console.log('helloだね'); // "hello" を入力したとき
} else {
    console.log('helloじゃないよ'); // それ以外のとき
}
```
</CodePreview>

### if...else文のQuickCheck

<QuickCheck>

次のコードでは何が表示されますか？

```javascript

if (input === 'hello') {
    console.log('helloだね');
} else {
    console.log('helloじゃないよ');
}
```

<Hint>
`input === 'hello'` が `false` の場合、`if` と `else` のどちらが実行されるか確認しましょう。
</Hint>

<Answer>
`helloじゃないよ` と表示されます。

条件が `false` なので、`else` の波括弧内が実行されます。
</Answer>

</QuickCheck>

## else if 文

「〇〇のとき」→「そうじゃなくて△△のとき」→「それ以外のとき」のように、**3つ以上の分岐**を作りたい場合は `else if` を使います。

```js
if (条件1) {
    // 条件1 が true のときの処理
} else if (条件2) {
    // 条件1 が false かつ 条件2 が true のときの処理
} else {
    // どの条件も false のときの処理
}
```

<CodePreview>
```javascript
let number = Number(prompt('数値を入力して下さい'));

if (number >= 100) {
    console.log('100以上');
} else if (number >= 50) {
    console.log('50以上100未満');
} else {
    console.log('50未満');
}
```
</CodePreview>

### else if文のQuickCheck 1

<QuickCheck>

次のコードでは何が表示されますか？

```javascript
let number = 75;

if (number >= 100) {
    console.log('100以上');
} else if (number >= 50) {
    console.log('50以上100未満');
} else {
    console.log('50未満');
}
```

<Hint>
上から順番に条件を確認し、最初に `true` になる条件を探しましょう。
</Hint>

<Answer>
`50以上100未満` と表示されます。

`number >= 100` は `false` ですが、`number >= 50` は `true` です。そのため、2つ目の処理が実行されます。
</Answer>

</QuickCheck>

### else if文のQuickCheck 2

<QuickCheck>

次のコードでは何が表示されますか？

```javascript
let number = 120;

if (number >= 50) {
    console.log('50以上');
} else if (number >= 100) {
    console.log('100以上');
}
```

<Hint>
両方の条件が `true` になる場合でも、`else if` は上から順番に判定します。
</Hint>

<Answer>
`50以上` と表示されます。

最初の `number >= 50` が `true` になった時点で、その処理が実行されます。後ろの `else if` は判定されません。
</Answer>

</QuickCheck>
